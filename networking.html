There are several protocols to cover. This table lists each one and describes their purpose.

Protocol

What is stands for

What is it used for?
TCP
 Transmission Control Protocol Responsible for splitting data into packets and reassembling the packets back into the original data once they reach the receiving device, as well as checking that data is correctly send and delivered.
IP
 Internet Protocol Responsible for directing packets to their destination across the network.
HTTP
 Hyper Text Transfer Protocol Used by web browsers to access websites and communicate with web servers.
HTTPS
 HTTP Secure A more secure version of HTTP. Encrypts all information sent and received.
FTP
 File Transfer Protocol Used to access, edit and move files between devices on a network.
POP3
 Post Office Protocol - version 3 Used to retrieve emails from a server. The server holds the email until you download it, and it is then deleted from the server.
IMAP
 Internet Message Access Protocol Used to retrieve emails from a server. The server holds the email until you delete it - you only download a copy. Used by most web-based email clients like gmail.
SMTP
 Simple Mail Transfer Protocol Used to send emails. Also used to transfer emails between servers.

Network protocols are divided into layers

  • A layer is a group of protocols which have similar functions.
  • They are self-contained, meaning protocols in each layer don't need to know what's happening in other layers.
  • Each layer serves the layer above it - it does the hidden work needed for an action on the layer above.

Below is the example 4-layer model of network protocols

Layer

What protocols this layer covers

Protocol examples
Application layer Turning data into websites and other applications, and vice versa. HTTP, FTP, SMTP
Transport layer Splitting data into packets and checking data is sent and delivered. TCP
Network layer Making connections between networks and directing data. IP
Data link layer Passing data as electrical signals over the physical network. Ethernet

There are several advantages to the use of network layers:

  • It breaks network communication into manageable pieces, helping developers concentrate on only one area of the network without having to worry about the others.
  • As they are self-contained, they can efficiently be changed without the other layers being affected.
  • Having standards for each layer forces companies to make compatible, universal hardware hardware and software, so different brands will and always will work with each other in the same way.
